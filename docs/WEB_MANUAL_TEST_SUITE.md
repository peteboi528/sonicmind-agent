# Web 端手动测试集

这是一套面向 `frontend` Web 页面的手动测试集，目标不是穷举，而是用最少的测试覆盖最关键的回归风险、Agent 能力展示点和面试演示路径。

适用范围：

- 对话页 `Chat`
- 今日推荐 `Daily`
- 发现页 `Discover`
- 品味实验室 `Taste Lab`
- 音乐画像 `Profile`
- 我的库 `Library`
- 歌单页 `Playlist`
- 侧边栏 / 网易云登录 / 导入 / 偏好训练

建议执行节奏：

1. 先跑 `P0-P1`，确认服务可用、基本链路没坏
2. 再跑 `P2-P5`，确认推荐、发现、记忆、画像链路正常
3. 最后跑 `P6`，专门验证新加的 agent tool

建议记录方式：

- 每条用 `Pass / Fail / Blocked`
- 失败时记录：输入文案、页面位置、最终响应、控制台报错、网络请求状态码
- 对涉及推荐结果的 case，截图保留首屏和最终答案

## P0 冒烟测试

### WEB-SMOKE-01 首屏可用
- 目标：确认 Web 能正常加载
- 前置条件：后端和前端都已启动
- 步骤：
1. 打开首页
2. 观察顶部 tab、侧边栏按钮、播放器区域是否渲染
- 预期结果：
1. 页面无白屏
2. 顶部至少可见 `对话 / 今日 / 发现 / 实验室 / 画像 / 我的库 / 历史 / 歌单`
3. 控制台无首屏致命报错

### WEB-SMOKE-02 Tab 切换不丢状态
- 目标：验证 `KeepAlive` 缓存和页面切换稳定性
- 步骤：
1. 进入 `对话` 输入一条消息但先不要发送
2. 切到 `发现`、`画像`、`歌单`
3. 回到 `对话`
- 预期结果：
1. 未发送输入仍保留
2. 页面切换无明显闪退或整页重载感

### WEB-SMOKE-03 侧边栏开关稳定
- 目标：验证抽屉和主界面切换
- 步骤：
1. 点击左上角菜单按钮打开侧边栏
2. 点击遮罩关闭
3. 再次打开并点击侧边栏关闭按钮
- 预期结果：
1. 侧边栏打开/关闭动画正常
2. 关闭后主界面仍可继续交互

## P1 核心对话链路

### WEB-CHAT-01 基础推荐
- 目标：验证聊天流式返回和推荐卡片渲染
- 输入：`推荐几首适合深夜的歌`
- 预期结果：
1. 先出现思考中或工具执行中的过程提示
2. 最终返回文本回答 + 歌曲卡片
3. 歌曲卡片可播放、可做后续动作

### WEB-CHAT-02 专辑讲解
- 目标：验证知识型回答和专辑信息展示
- 输入：`讲讲 Blonde 这张专辑，乐评怎么说？`
- 预期结果：
1. 输出不是泛泛一句话
2. 应包含专辑解读、评论或总结信息
3. 如有专辑卡或相关条目，应能正常展开/播放

### WEB-CHAT-03 艺人路线分析
- 目标：验证艺人知识链路
- 输入：`The Weeknd 的音乐路线是什么？`
- 预期结果：
1. 回答应围绕风格变化、阶段演进或代表时期
2. 不应只给一串歌曲名

### WEB-CHAT-04 长回答不中断
- 目标：验证流式答案在较长输出下不会截断
- 输入：`详细比较 Frank Ocean 和 The Weeknd 的音乐风格差异`
- 预期结果：
1. 过程消息持续更新
2. 最终答案完整收束，没有半句停住

### WEB-CHAT-05 延续指令可工作
- 目标：验证多轮上下文
- 步骤：
1. 先输入：`推荐几首适合深夜的歌`
2. 再输入：`再来一点，别和刚才重复`
- 预期结果：
1. 第二轮仍识别为延续推荐
2. 结果和上一轮明显有差异
3. 不应原样重复首轮候选

### WEB-CHAT-06 历史刷新后仍在
- 目标：验证本地会话持久化
- 步骤：
1. 发送 2 到 3 轮消息
2. 刷新浏览器
- 预期结果：
1. 聊天记录仍在
2. 线程上下文没有完全丢失

## P6 新增 Agent Tool 定向测试

这一组建议全部在 `对话` 页执行，因为这些能力当前主要以 chat tool 形式接入。

### WEB-TOOL-01 music_compare
- 输入：`比较 Drake 和 Future 的风格差异，并给我各自的入门歌`
- 预期结果：
1. 回答应明确是“比较”而非单方介绍
2. 最好能看到比较维度，例如风格、情绪、节奏、代表作品
3. 若结果里出现推荐曲，应和两位艺人对应

### WEB-TOOL-02 concert_events
- 输入：`最近上海有什么 The Weeknd 相关演出消息吗`
- 预期结果：
1. 应返回演出/巡演信息或诚实说明未查到
2. 不应伪造具体场馆和日期
3. 若有来源，最好能体现来源名或链接

### WEB-TOOL-03 playlist_repair
- 前置条件：先生成一个风格比较杂的歌单，或让上一轮推荐结果尽量混搭
- 输入：`帮我看看这个歌单有什么问题，顺便修一下思路`
- 预期结果：
1. 应指出具体问题，如重复、风格跳跃、能量断层、语言混杂
2. 应给出修复建议，而不是只说“挺不错的”
3. 没有上下文时，应诚实提示缺少待分析对象

### WEB-TOOL-04 taste_shift_detector
- 前置条件：当前用户已经有一段时间的听歌、评分或实验反馈历史
- 输入：`最近 30 天我的口味变化大吗`
- 预期结果：
1. 应体现“近期 vs 历史”的对比
2. 应指出上升风格、下降风格、或新出现的艺人/情绪
3. 数据不足时应诚实说明

### WEB-TOOL-05 music_fact_check
- 输入：`帮我核验一下：Blonde 是 Frank Ocean 在 2016 年发行的专辑`
- 预期结果：
1. 应给出核验结论
2. 应区分“已验证”和“不确定”
3. 不应在证据不足时强行下判断

### WEB-TOOL-06 music_fact_check 反幻觉
- 输入：`帮我核验一下：Drake 是 2024 年发行《Blonde》的人`
- 预期结果：
1. 应识别为错误或至少不成立
2. 不应顺着错误陈述扩写

### WEB-TOOL-07 recommend_explainer
- 步骤：
1. 先输入：`推荐几首适合深夜的歌`
2. 再输入：`解释一下你为什么推荐这些歌`
- 预期结果：
1. 第二轮应解释上一轮推荐
2. 应尽量按歌曲逐首解释，或至少给出全局理由
3. 不应生成和上一轮无关的新推荐替代解释

### WEB-TOOL-08 recommend_explainer 空上下文
- 输入：`解释一下你刚才推荐这些歌的原因`
- 前置条件：当前会话里还没有推荐结果
- 预期结果：
1. 应诚实提示先给出推荐再解释
2. 不应编造并不存在的“刚才推荐”

### WEB-TOOL-09 continue 去重回归
- 步骤：
1. 输入：`推荐和 Drake、Future、Molly Santana 接近的歌手`
2. 再输入：`再来一点`
- 预期结果：
1. 第二轮不应原样重复上一轮 6 个歌手
2. 若数量不足，应明确说明而不是假装翻页

### WEB-TOOL-10 空结果恢复
- 输入：`推荐一些特别冷门但和我口味贴近的火星语电子民谣`
- 预期结果：
1. 若首轮查不到，不应立刻僵死
2. 应出现换个思路、重试或诚实降级的行为
3. 最终不应长时间卡在思考中

## P7 组合式长期用户旅程

这部分适合一次性完整演练，也最适合面试前自测。

### WEB-JOURNEY-01 新用户冷启动
- 步骤：
1. 切换到全新 userId
2. 在侧边栏写入偏好：`我喜欢梦幻、氛围感强的 R&B`
3. 去 `今日` 获取推荐
4. 去 `发现` 搜 `The Weeknd`
5. 回 `对话` 输入：`推荐点类似但更冷门的`
- 预期结果：
1. 新用户能从空状态走到可推荐
2. 推荐逐渐体现已学习偏好

### WEB-JOURNEY-02 有库用户闭环
- 步骤：
1. 导入一个网易云歌单或手动入库 5 到 10 首
2. 在 `我的库` 给几首打分
3. 去 `画像` 刷新
4. 去 `实验室` 做一轮 taste experiment
5. 回 `对话` 让系统解释推荐原因
- 预期结果：
1. 评分、记忆、画像、推荐解释之间有闭环感
2. 页面之间的数据传递应基本一致

## P8 库搜索 / 历史 tab（2026-06-30 新增）

### WEB-LIB-01 库内搜索
- 目标：长库里快速找到自己的歌
- 前置条件：库里已入库若干曲目（不同歌手/标签）
- 步骤：
1. 进入「我的库」
2. 在搜索框输入某首歌名片段 / 歌手名 / 专辑名 / 标签名
3. 点 × 清除搜索
- 预期结果：
1. 列表实时按输入过滤（歌名/歌手/专辑/标签任一命中，不区分大小写）
2. 搜索可与标签筛选叠加（AND）
3. 无匹配时提示「没有匹配「xxx」的曲目」并给清除入口
4. 清除后恢复全量；「播放全部」数量随结果变化

### WEB-HIST-01 最近对话列表
- 目标：回看历史会话

- 步骤：
1. 进入「历史」tab，停在「最近对话」分段
- 预期结果：
1. 列出历史线程，显示标题（无标题用首条用户消息）、消息条数、相对时间
2. 空库时提示「还没有历史对话」

### WEB-HIST-02 点开历史对话跳转载入
- 目标：从历史 tab 进任一会话
- 步骤：
1. 在「最近对话」点任一线程
- 预期结果：
1. 自动切到「对话」tab
2. 该线程的消息载入聊天窗（含原卡片）

### WEB-HIST-03 听歌记录
- 目标：回看最近听过什么
- 前置条件：本次更新后播放过几首（本地 + 在线各一，听完 / 秒跳各一次）
- 步骤：
1. 进入「历史」tab，切到「听歌记录」分段
- 预期结果：
1. 列出最近播放，最近在前
2. 每条显示封面 / 标题-歌手 / 完整收听·秒跳·听了Nm / 相对时间
3. 本地曲与在线曲都能显示标题（在线曲标题来自播放时上报的元数据）
4. 点击可播曲目 → 入播放器；无元数据的旧在线记录置灰不可点
5. 空时提示「还没有听歌记录，播放几首试试」
- 备注：本次更新前播放的在线曲（未上报元数据）会置灰，属预期

### WEB-HIST-04 清空历史对话
- 步骤：在「最近对话」点「清空全部」并确认
- 预期结果：列表清空，不影响听歌记录与曲库
